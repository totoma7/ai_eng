# 11장 (심화). Human-in-the-Loop — 에스컬레이션 판단력의 과학

> 4장 가드레일의 "휴먼 체크포인트", 10장의 "서킷 브레이커"를 뒤집어 본다 —
> **에이전트는 언제 사람에게 물어야 하는지 스스로 아는가?**

## 11.1 HiL-Bench — "물어볼 줄 아는 능력"의 측정 (2026.4)

첫 체계적 측정 벤치마크. 설계가 영리하다:

1. **잘 명세된 과업**을 가져와서
2. **현실적인 블로커**(누락된 권한, 모호한 요구사항, 충돌하는 제약)를 주입하고
3. 에이전트에게 `ask_human()` 도구를 준다

결과:

| 조건 | 성능 |
|------|------|
| 완전한 정보 제공 시 | **~90% pass@3** |
| 블로커 주입 시 | **유의미한 성능 급락** |

해석: 과업 수행 능력과 **"막혔음을 인지하고 에스컬레이션하는 능력"은 별개의 역량**이다.
모델은 모르는 것을 모른 채 밀고 나간다 — 8장의 "승리 선언 편향"과 같은 뿌리.

## 11.2 루프 엔지니어링 시사점

`ask_human()`은 도구 목록에 추가하는 것으로 끝나지 않는다. 하네스/루프 차원의 설계가 필요하다:

| 설계 지점 | 질문 |
|----------|------|
| 트리거 조건 | 무엇이 "막힘"인가? (무진행 감지와 연동 — 4장) |
| 비용 비대칭 | 잘못된 질문(사람 시간 낭비) vs 잘못된 추측(잘못된 코드) — 어느 쪽이 비싼가? |
| 질문 품질 | "어떻게 할까요?"가 아니라 "A/B 중 선택 + 각각의 트레이드오프" 형식 강제 |
| 타이밍 | 동기(루프 중단) vs 비동기(트리아지 인박스에 적재 — 4장 오케스트레이션 루프) |

ralph-loop 실습과의 연결: `exit 2`(무진행)와 `exit 1`(상한 도달)이 바로 **암묵적 에스컬레이션**이다 —
루프가 자기 한계를 인지하고 사람에게 공을 넘기는 가장 원시적인 형태.

## 11.3 승인 게이트를 넘어서 — 시간 축의 HITL

LangChain의 2026년 4월 가이드의 핵심 전환:

> 사람의 입력은 실행 시점의 **승인 게이트만이 아니다.**
> 프롬프트·도구·메모리·평가자를 **시간에 걸쳐 개선하는 감독**이기도 하다.

두 시간 축으로 정리하면:

| 축 | 역할 | 메커니즘 |
|----|------|---------|
| **실행 중** (runtime) | 고위험 행동 차단 | 승인 게이트, 서킷 브레이커, ask_human() |
| **실행 사이** (design-time) | 시스템 자체의 개선 | 실패 사례 → 골든 데이터셋(9장) → 프롬프트/도구/평가자 수정 |

두 번째 축이 루프 엔지니어링의 본령과 만난다: **사람이 루프의 매 턴이 아니라, 루프를 설계하고 개선하는 메타 루프에 위치한다.** 1장의 "에이전트에게 프롬프트하는 사람을 대체한다"는 말의 정확한 의미 — 사람이 사라지는 게 아니라 **한 층 위로 올라간다.**

## 11.4 in-the-loop / on-the-loop / above-the-loop

| 위치 | 사람의 역할 | 적합 상황 |
|------|-----------|----------|
| **in-the-loop** | 매 단계 승인 | 고위험·저빈도 작업, 초기 신뢰 구축 |
| **on-the-loop** | 감독 + 서킷 브레이커 개입 (10장) | 프로덕션 표준 |
| **above-the-loop** | 루프 자체의 설계·개선 (메타 루프) | 루프 엔지니어링의 목표 지점 |

5장의 경고와 연결: above-the-loop로 올라가되 **인지적 항복은 금물** — 위로 올라간다는 건 출력 읽기를 그만둔다는 뜻이 아니다.

## 11.5 요약

1. "물어볼 줄 아는 능력"은 독립 역량이고, 측정 가능해졌다 (HiL-Bench).
2. `ask_human()`은 도구가 아니라 **하네스 설계 문제** — 트리거·비용 비대칭·질문 품질·타이밍.
3. HITL은 승인 게이트(실행 중) + 시스템 개선 감독(실행 사이)의 **이중 시간 축**.
4. 루프 엔지니어링에서 사람의 최종 위치는 above-the-loop — 단, 눈은 루프 안에.
