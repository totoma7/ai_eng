# 12장 (심화). 환경 엔지니어링 — AI가 읽을 수 있는 시스템 만들기

> 차세대 프런티어. 하네스 오케스트레이션이 클라우드 표준 인프라로 범용화되면서,
> 경쟁 우위는 **"환경 자체를 AI에게 읽기 쉽게(legible) 재설계하는 것"** 으로 이동 중이다.

## 12.1 발상의 전환

지금까지의 방향: 적대적인 시스템을 헤쳐나갈 **더 똑똑한 에이전트** 만들기.
환경 엔지니어링의 방향: 에이전트가 헤맬 필요 없는 **더 읽기 쉬운 환경** 만들기.

> 내부 API, 코드베이스, DB를 AI에게 본질적으로 legible하게 재아키텍팅하라.

레이어 스택(6장)에 한 층이 추가되는 셈이다:

```
루프 엔지니어링        (언제·왜·언제까지)
하네스 엔지니어링      (무엇을 갖고·어디까지)
환경 엔지니어링        (어떤 세계 안에서)   ← NEW
모델                  (추론 능력)
```

## 12.2 Orientation Tax와 Grep-Spree

문제의 이름:

- **Orientation Tax (정위 세금)**: 에이전트가 환경의 구조를 모를 때 "여기가 어디지?"를 파악하는 데 낭비되는 토큰
- **Grep-Spree (그렙 폭주)**: 디렉토리를 맹목적으로 검색하며 토큰을 태우는 실패 모드 — 4장의 "침묵 실패"의 친척

해법 — **구조적 코드 매핑 (Structural Code Mapping)**:
tree-sitter, RepoMapper 같은 도구로 **토큰 효율적인 환경 지도**를 에이전트에게 제공한다.
사람에게 건물 평면도를 주듯, 에이전트에게 저장소 지도를 준다.

## 12.3 환경 성숙도 — Observable Runtime이 임계점

환경 성숙도 모델에서 **Level 3 (Observable Runtime)** 이 결정적 문턱이다:

- 세션 체크포인팅
- 파일시스템을 통한 명시적 메모리 (`AGENTS.md`, `todo.md` 로그)
- **환경이 에이전트를 올바른 실행으로 유도**한다

낯익지 않은가 — 4장의 외부 메모리, ralph-loop 실습의 `progress.md`가 정확히 이것이다.
환경 엔지니어링은 우리가 루프 차원에서 해온 일을 **시스템 전체 차원으로 일반화**한 것.

## 12.4 실무 체크리스트 — 내 시스템은 AI-legible한가?

| 영역 | 질문 | 개선 예시 |
|------|------|----------|
| 코드베이스 | 에이전트가 지도 없이 구조를 파악할 수 있나? | AGENTS.md/CLAUDE.md, 저장소 지도, 일관된 디렉토리 컨벤션 |
| API | 에러 메시지가 "다음 행동"을 알려주나? | "401 Unauthorized" → "토큰 만료. POST /auth/refresh로 갱신" |
| DB/스키마 | 용어가 코드·문서·대화에서 일치하나? | 도메인 용어 사전, 스키마 주석 |
| 테스트 | 실패 출력이 원인을 가리키나? | 단언 메시지에 기대값/실제값/맥락 포함 |
| 빌드/스크립트 | 한 명령으로 검증 가능한가? | `make verify` 같은 단일 진입점 — **게이트로 쓰기 좋은 환경** |
| 문서 | 사람용 문서와 별개로 기계가 파싱할 구조가 있나? | 구조화된 메타데이터, 예제 중심 문서 |

핵심 통찰: **legible한 환경은 게이트 만들기도 쉽다.** 검증 가능한 목표 명세(2장)가 어려운 시스템은 대개 환경이 불투명한 시스템이다.

## 12.5 자기 개선 하네스 (Self-Improving Harness)

초기 단계 연구 방향 — 에이전트가 **자기 하네스를 스스로 개선**한다:

- 멀티 에이전트 제안자(proposer)가 하네스 변경안 생성
- 평가 기반 선택(9장의 evals가 심판)
- git worktree 격리(4장)에서 안전하게 실험
- Stanford/MIT의 Meta-Harness: **자동 하네스 최적화가 수작업 엔지니어링을 능가**함을 실증

루프 엔지니어링의 자기 적용이다 — 루프를 개선하는 루프. 11장의 above-the-loop에서 사람이 하던 메타 루프 작업의 일부가 다시 자동화되는 흐름. 단, 5장의 경고가 제곱으로 적용된다: 하네스를 고치는 에이전트의 출력을 읽지 않으면, 이해 부채가 인프라 레벨에 쌓인다.

## 12.6 전체 그림 — 학습 노트 총정리

```
            사람 (above-the-loop, 메타 루프)            ← 11장
              ↓ 설계·개선
┌───────────────────────────────────────────┐
│ 루프: 트리거 + 게이트 + 가드레일 + 외부 메모리   │ ← 1~5장, 실습
│   ├ 안쪽: 도구 호출 루프 (프레임워크)           │ ← 7장
│   ├ 토폴로지: 수퍼바이저 + PEV                 │ ← 10장
│   └ 게이트의 품질: Evals                      │ ← 9장
├───────────────────────────────────────────┤
│ 하네스: 5계층 (도구·검증·컨텍스트·가드레일·관측)  │ ← 6, 8장
├───────────────────────────────────────────┤
│ 환경: AI-legible 시스템                       │ ← 12장
├───────────────────────────────────────────┤
│ 모델                                        │
└───────────────────────────────────────────┘
```

한 문장 요약: **모델은 주어진다. 루프·하네스·환경이 엔지니어링이다.**
