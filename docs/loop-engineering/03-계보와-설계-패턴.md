# 3장. 계보와 설계 패턴

루프 엔지니어링은 갑자기 나온 게 아니라 4년에 걸친 진화의 산물이다.

## 3.1 역사적 계보

```
ReAct (2022.10)  →  AutoGPT (2023.3)  →  Ralph Loop (2025.7)  →  /goal (2026 봄)  →  루프 엔지니어링 (2026.6~)
   학술 기반          최초 대중 실증          컨텍스트 리셋 혁신        네이티브 구현         체계화된 실천법
```

| 단계 | 시기 | 핵심 기여 | 한계 |
|------|------|----------|------|
| **ReAct** | 2022.10 (Princeton + Google Research) | 추론 흔적(reasoning trace)과 구체적 행동을 결합하는 패턴. 대부분의 프로덕션 에이전트의 기반. ALFWorld 과업 34% 개선 | 단일 스텝 중심 |
| **AutoGPT** | 2023.3 | 목표를 하위 작업으로 분해 + 도구 접근(웹, 파일). 개발자 수요를 최초로 실증 | 무한 루프, 감당 불가능한 비용 |
| **Ralph Loop** | 2025.7 (Geoffrey Huntley) | **컨텍스트 오버플로 해법**: 반복마다 LLM 컨텍스트 윈도를 리셋하고, 상태는 파일시스템에 유지. 에이전트가 코드를 고치고 종료하면 셸 스크립트가 새 컨텍스트로 재시작 | 단일 에이전트, 수동 검증 |
| **/goal** | 2026 봄 | Claude Code / Codex CLI 네이티브 구현. 별도 평가자 모델이 완료 조건을 검증 | — |
| **루프 엔지니어링** | 2026.6~ | 목표 명세 + 트리거 선택 + 체계적 가드레일로 **신뢰할 수 있는 루프를 설계하는 규율** | 진행 중 |

현재 흐름: 단일 에이전트 루프(ralph류)는 이미 "올드한" 것으로 취급되고, **멀티 에이전트 감독(supervision)** 이 새 레이어로 부상 중.

## 3.2 세대별 설계 패턴

### 학술 세대 (2022–2023)

- **ReAct** — 추론 + 행동 결합. 단일 스텝 과업의 기본기.
- **Reflexion** — 자기평가 레이어 추가. 시행착오형 과업에 유리하나 비용 증가.
- **Plan-and-Execute** — 계획과 실행을 분리. 병렬화로 최대 3.6배 속도 향상.

### 아키텍처 세대 (2024)

- **OODA Loop** — 관찰과 결정 사이에 Orient(상황 정위) 단계 추가. 동적 환경에 적합.
- **Inner/Outer 이중 루프** — 안쪽 루프(스텝 실행)와 바깥 루프(전략 감독)를 분리. 안쪽이 정체되면 바깥이 전략을 리셋.
- **멀티 에이전트 오케스트레이션** — 수퍼바이저가 전문 서브에이전트들을 조정. 토큰 약 **15배** 소모.

### 실무자 세대 (2025–2026)

- **Ralph Loop** — 파일시스템 상태 유지 + 컨텍스트 리셋.
- **/goal** — 평가자 모델 기반 완료 검증.
- **오케스트레이션 루프** — 아침마다 자동 트리아지 → worktree 격리 병렬 작업 → 자동 PR (4장 참조).

## 3.3 패턴 선택 가이드

| 상황 | 권장 패턴 |
|------|----------|
| 단일 스텝 과업 | ReAct |
| 자기교정이 필요한 멀티 스텝 | ReAct + Reflexion |
| 긴 코드베이스 작업 | Ralph Loop 또는 /goal |
| 병렬 탐색 | 멀티 에이전트 오케스트레이션 |
| 의존성이 이미 알려진 작업 | Plan-and-Execute |
| 동적 환경 | OODA |

## 3.4 비용 감각

- 단일 에이전트 루프: 일반 채팅 대비 약 **4배** 토큰.
- 멀티 에이전트: 약 **15배**.
- 문서화된 사고 사례: 5분 동안 중복 도구 호출 400회.
- 결론: **비용의 무게중심이 '프롬프트 작성'에서 '루프 관리'로 이동했다.** 반복 상한, 무진행 감지, 달러 예산이 필수. (→ 4장)
